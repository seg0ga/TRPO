## Описание работы

1. Разработать любой правильно (корректно) работающий функционал согласно своему заданию.
    - язык разработки C/C++;
    - вся разработка в git, со всеми промежуточными коммитами;
    - реализация должна покрывать все варианты использования;
    - реализация должна поддерживать типы одинарной и двойной точности;
    - код может быть последовательным (не параллельным);
    - допускается реализация только на одной архитектуре (win, linux, macos).
2. Скачать дистрибутив OpenBLAS (https://github.com/OpenMathLib/OpenBLAS)
3. Создать набор тестов сравнивающих производительность вашей реализации по сравнению с OpenBLAS:
    - подобрать размер, чтобы задача решалась минимум 1 мин на тестирующем железе;
    - запуск функционала 10 раз с вычислением только чистого времени счета, без учета времени создания входных массивов;
    - запуск теста на 1,2,4,8 и 16 потоках;
    - вычисление относительной производительности в процентах от OpenBLAS для каждого запуска теста;
    - вычисление среднего геометрического производительности по 10 запускам;
    - +20 баллов за параллельную реализацию, не уступающую более 30% производительности OpenBLAS.
4. Оформить отчет, содержащий:
    - титульный лист;
    - содержание;
    - постановку задачи;
    - ссылку на репозиторий с функционалом и тестами;
    - скриншоты запуска и результатов тестирования;
    - сводную таблицу тестирования производительности.

### Запуск программ:
Для теста OpenBLAS c двойной точностью:
```
g++ -O3 -march=native -fopenmp test_gemm_openblas_double.cpp -o test_gemm_openblas -lopenblas -lpthread
./test_gemm_openblas
```
Для теста OpenBLAS c одинарной точностью:
```
g++ -O3 -march=native -fopenmp test_gemm_openblas_float.cpp -o test_gemm_openblas -lopenblas -lpthread
./test_gemm_openblas
```
